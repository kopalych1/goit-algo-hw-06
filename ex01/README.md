Path from 1 -> 8
BFS: [1, 6, 8]
DFS: [1, 2, 3, 4, 5, 8]

Path from 5 -> 9
BFS: [5, 8, 6, 9]
DFS: [5, 4, 3, 1, 6, 9]

Path from 6 -> 5
BFS: [6, 8, 5]
DFS: [6, 1, 2, 3, 4, 5]


З результатів видно, що DFS не завжди знаходить найкоротший шлях через специфіку свого алгоритму: 
DFS йде вглиб по першій знайденій гілці до кінця, і тільки потім повертається для перевірки 
інших варіантів. Це призводить до того, що перший знайдений шлях може бути не оптимальним.

BFS є кращим вибором для пошуку найкоротшого шляху, оскільки він досліджує граф пошарово 
(рівень за рівнем), що гарантує знаходження шляху з мінімальною кількістю вершин. 
Це особливо важливо для зациклених графів, де існує багато різних маршрутів між вершинами.

**Висновок:** Для пошуку найкоротшого шляху в незваженому графі завжди використовуйте BFS, 
а DFS підходить для завдань обходу графу, пошуку зв'язності або топологічного сортування.
